초대 링크(?invite=CODE)로 진입 → 가입

피초대자가 초대 링크를 열면 URL의 ?invite= 코드가 즉시 로컬에 임시 저장되고, 닉네임만으로 가입이 끝나는 온보딩을 그대로 진행한다. 가입 완료 후 홈 화면에 진입.

4
분기

마이페이지 "초대코드 입력" — 자동 등록 vs 수동 입력

가입 경로에 따라 마이페이지 회원정보의 '초대코드 입력' 행이 다르게 시작한다. 링크로 들어온 경우 서버에서 즉시 자동 등록되어 잠기고, 코드 없이 가입한 경우 입력란이 열려 있어 1회 직접 입력해야 한다. 두 경로 모두 최종 등록 후에는 같은 잠금 화면으로 수렴한다.

자동 등록 · 딥링크

?invite=CODE 로 진입한 경우

가입 직후 서버가 대기 중인 코드를 자동으로 매칭·등록. 마이페이지 행이 처음부터 "입력 완료"로 표시되고, 입력란은 비어 있는 채로 잠긴다.

마이페이지 행: "입력 완료" (자동)
입력란 비어있음 + 잠김 · "이미 초대코드를 입력했어요"
수동 입력 · 코드 없이 가입

초대 없이 그냥 가입한 경우

딥링크 없이 가입하면 입력란이 활성 상태로 열려 있다. 친구에게 나중에 코드를 받아 8자 이내로 직접 입력하고 '등록하기'를 눌러야 한다.

입력란 활성 · 빈 상태 (예: AB23CD)
직접 입력·등록 후 잠김 (입력값 유지된 채 비활성화)
코드 입력은 계정당 평생 1회. 자동/수동 모두 users.invited_by 1회 설정 후 불변 — 재입력·변경 불가.

첫 구매 성사

피초대자 첫 구매 완료 → 양쪽 500P 지급

피초대자의 첫 구매가 성사 처리되면 초대자에게는 그 자리에서 즉시 500P가 지급된다. 피초대자 몫 500P는 별도 배치 없이, 피초대자가 다음에 앱에 접속하는 순간(홈/마이페이지 진입 시) 자동으로 지급되고 활동내역에 기록된다.

초대자 측: 성사 처리 완료 토스트 · 즉시 500P 반영
초대자 홈: 500P → 1,000P (초대 보상 반영)
피초대자 측: 활동내역에 "친구초대 보상(피초대자) +500P" 자동 기록

참고 사항

  • 모든 화면은 로컬 프리뷰 서버(app 포트 3366/3377/3388/3399, cms 포트 3333)에서 실제 계정 생성 → 초대코드 교환 → 구매 성사 처리 → 활동내역 확인까지 실행한 뒤 캡처했다.
  • 자동 등록과 수동 입력은 최종적으로 동일한 invited_by 필드 하나로 수렴하며, UI상 구분되는 지점은 잠금 직전(입력란 활성 여부)뿐이다.
  • CMS 대시보드의 집계 수치는 이 문서 캡처 시점에 실제로 발생한 테스트 초대 건이 반영된 실데이터다.
