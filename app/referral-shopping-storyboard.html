홈 "쇼핑하고 티켓 받기" 섹션

홈 탭 중단의 쇼핑몰 섹션. "바로가기로 들어가 구매하면 티켓이 쌓여요" 카피와 함께 쿠팡(BEST)·알리익스프레스·테무 카드가 노출된다. 카드의 "바로가기 ›"는 이동 브릿지로 직행하고, "전체 ›"는 적립 가능 쇼핑몰 목록으로 진입한다.

적립 가능 쇼핑몰

"전체 ›"로 진입하는 제휴몰 전체 목록. 종합 쇼핑몰 4개(쿠팡·11번가·G마켓·옥션) + 해외직구 3개(알리익스프레스·테무·아이허브). 쇼핑몰명을 누르면 적립 안내로, "바로가기 ›"를 누르면 안내를 건너뛰고 바로 이동 브릿지로 간다.

앱 · 적립 안내

쿠팡 적립 안내 — "구매 시 랜덤 티켓 1장" + 3단계

쇼핑몰별 적립 안내 화면. 로고 카드에 "구매 시 랜덤 티켓 1장", CTA "쿠팡 바로가기" 아래에 3단계 안내(① 바로가기로 쿠팡 접속 ② 24시간 이내 구매 ③ 티켓 확정)가 배치된다. 홈 화면 바로 가기 추가 버튼(빨간색)도 이 화면의 표준 위치.

쿠팡 적립 안내 · 3단계 안내 포함
SoT 확정 · 화면 미구현: 3단계 중 "티켓 확정" 문구가 "구매 확정 후 7일 사용 가능"으로만 표기돼 있다. 마스터 정책서(policyspec, SoT)는 연동 D+7 / 미연동 D+30으로 명확히 확정돼 있고 CMS·적립 완료 화면은 이를 정확히 반영한다. 이 화면만 미연동 D+30 표기가 빠진 구현 누락이며 결정을 기다리는 사항이 아니다 — 정정 필요.

시스템 · 브릿지

이동 브릿지 — 적립 추적 시작

"쿠팡 바로가기"를 누르면 이동 브릿지가 뜬다. "쿠팡으로 이동 중" + 체크리스트(적립 추적 시작 · 24시간 유효)로 경유 추적이 시작됐음을 알리고, 실서비스에서는 이 시점에 쿠팡 앱/웹으로 전환된다. 클릭 로그(click_logs)가 이 시점에 기록된다.

이동 브릿지 · "쿠팡으로 이동 중" + 24시간 유효

데모 시뮬

구매 완료 — 금액 입력 시뮬 (데모 전용 장치)

실서비스에서는 외부 쇼핑몰에서 구매하면 제휴몰 포스트백(구매 확인 콜백)이 서버로 수신된다. 데모에서는 이 포스트백을 대신하는 시뮬 바텀시트가 브릿지 3.5초 후 자동으로 떠서, 구매 금액을 입력하면(예: 53,000원) "티켓 받기"로 포스트백 수신·가지급 적립까지 실데이터로 처리된다(Firestore postbacks + user_tickets 기록).

데모 시뮬 바텀시트 · 53,000원 입력 → "티켓 받기"
정책 불일치(데모 장치) — 형님 결정 필요: 시트의 미리보기가 "적립 예정: 실버 1장"으로 등급을 사전 노출한다. 정책은 확정 전까지 등급 비공개(랜덤 표기). 데모 전용 장치라 실서비스와 무관하지만, 데모를 개발자·이해관계자에게 공유하는 용도라면 "랜덤 티켓 1장"으로 고칠지 결정 대기.

티켓 적립 완료 — 랜덤 티켓 1장 · 대기중

적립 완료 화면. 구매 정보(쿠팡 53,000원)와 "1장 적립됨" 배지, 그리고 "랜덤 티켓 1장 · 대기중" 안내가 노출된다. 확정 전에는 실제 등급(브론즈·실버·골드)을 공개하지 않는 정책이 화면에 그대로 반영돼 있고, 확정 대기 기간도 연동 D+7 / 미연동 D+30으로 정확히 표기된다. "홈으로 가기"로 복귀.

적립 완료 · 랜덤 티켓 1장 대기중 + D+7/D+30 안내
관찰: 데스크톱 폰 프레임(캡처 기준)에서는 정상 렌더되나, 좁은 모바일 뷰포트(390px)에서는 이 화면 하단(홈으로 가기 CTA)이 잘리는 앱 렌더링 이슈가 있다. 위 썸네일은 정상 렌더 상태의 실제 화면이며 합성·수정본이 아니다. 실기기 확인 필요 — 미결 보고 참조.

혜택 탭 — 대기 현황

구매 확정 처리 — 가지급 티켓 → 등급 티켓 전환

실서비스에서는 확정 대기 기간(연동 D+7 / 미연동 D+30) 경과 시 자동 확정되고, 데모에서는 CMS "데이터 관리(데모)"의 포스트백 수정에서 상태를 대기 → 확정으로 바꿔 그 시점을 재현한다. 확정 시 가지급 랜덤 티켓이 구매 금액 기준(Greedy 환산)의 등급 티켓으로 전환된다 — 53,000원은 실버 1장(브0·실1·골0).

포스트백 수정 모달 · 상태 "확정" 선택 → 저장 시 "가지급 티켓 1장 → 등급 티켓 전환"

① 쿠팡 적립 안내 3단계가 "확정 후 7일"만 표기 SoT 확정 · 미구현

적립 안내(스텝 3)의 "티켓 확정" 문구가 "구매 확정 후 7일 사용 가능"으로만 서술돼 미연동 D+30이 누락됐다. 이 캡처 계정은 미연동 게스트다. 마스터 정책서(SoT)는 연동 D+7 / 미연동 D+30으로 확정돼 있고, 같은 앱의 적립 완료 화면·혜택 탭·CMS는 모두 이를 정확히 표기한다 — 이 화면만 어긋난 구현 누락이며 형님 결정을 기다릴 사항이 아니다. 정정 필요. (적립 조건·제외 안내 ⑥ 보고 ③과 동일 건)

② 데모 구매 시뮬 시트가 등급 구성을 사전 노출 데모 한정

구매 금액 입력 시트(스텝 5)의 "적립 예정: 실버 1장" 미리보기가 등급을 확정 전에 노출한다. 정책은 확정 전까지 등급 비공개("랜덤 티켓"으로만 표시)다. 포스트백을 대신하는 데모 전용 장치라 실서비스와 무관하지만, 개발자·이해관계자 공유 시 오해 소지가 있어 "랜덤 티켓 1장"으로 고칠지 확인 필요. (⑤·⑥ 보고와 동일 건)

③ 적립 완료 화면이 좁은 모바일 뷰포트에서 하단 잘림 앱 렌더링

적립 완료 화면(pw-earn-done)이 데스크톱 폰 프레임에서는 정상이나 390px 모바일 뷰포트에서는 하단(홈으로 가기 CTA)이 잘린다. 위 썸네일은 정상 렌더 상태의 실화면이며 합성·수정본이 아니다. 실기기(iPhone) 재현 여부 확인 후 수정 대상.

④ 구매 적립 한도(일 5장·월 50장)가 앱 화면에 미노출 SoT 확정 · 안내 누락

일 5장 / 월 50장(랜덤 티켓=구매 건 기준, 2026-05-30 오너 결정으로 확정)이지만 이 플로우의 앱 화면 어디에도 한도가 표기되지 않는다. 형님 결정 대기가 아니라 안내 문구 추가가 필요한 항목 — 노출 위치는 "적립 조건·제외 안내" 스토리보드(⑥) 범위, 거기서도 동일하게 보고했다.

참고 사항

  • 캡처 환경: 배포본 pricepick-demo.vercel.app/app을 데스크톱 폰 프레임(390×844)에서 신규 게스트("다정한물개")로 실제 조작. CMS는 /cms firstadmin 계정. 캡처 일시 2026-07-09.
  • 등급 전환 기준(Greedy 환산): 골드 100,000 / 실버 50,000 / 브론즈 5,000원당 1장, 큰 단위부터. 티켓 1장 원환산 = 골드 2,000 / 실버 1,000 / 브론즈 100원(전체 보상률 2%). 53,000원 = 실버 1장(원환산 1,000원) 실측 확인.
  • 이 문서 캡처 과정에서 생성된 테스트 게스트 1명(다정한물개)과 확정 포스트백 1건은 데모 DB에 실데이터로 남아 있다(캡처 반복분 5건은 정리 완료).
  • 정책 근거는 마스터 정책서(policyspec.html — 2026-07-09 SoT 확정). 화면·다른 문서와 충돌하면 정책서가 우선한다.
