홈 미션 카드 — "쿠팡 구경" 진입

홈 상단 미션 카드 4종(쿠팡 구경 · 행운 룰렛 · 친구 초대 · 연속구매) 중 "쿠팡 구경 · 매일 100P" 카드를 탭하면 출석체크 화면으로 진입한다. 기존 출석체크는 정책상 "쿠팡 구경하기"로 대체됐다(경유 방문 습관화 + 30초 체류 검증으로 어뷰징 방지).

상단에 연속 구경 일수, 가운데에 이번 주 구경 현황(요일별 100P 체크)과 "5일 연속 달성까지" 진행바, 하단에 보상 안내(이벤트 티켓 1장)와 CTA "쿠팡 구경하러 가기"가 배치된다.

지급

적립 확정 — 매일 100P, 5일 연속이면 이벤트 티켓 1장 추가

체류 검증을 통과하면 그 자리에서 100P가 지급되고 토스트로 알린다. 5일 연속을 채운 날에는 달성 화면과 함께 이벤트 티켓 1장이 자동 지급되어 티켓함에 추가된다(경품 응모 전용). 구매가 아닌 무상 보상이므로 환불과 무관하며 회수가 없다.

5일 연속 달성 · 이벤트 티켓 +1장 지급 완료 + "100P가 적립됐어요" 토스트 (실제 지급 후 캡처)

CMS

운영자: 포인트 내역 — 출석 적립 원장 집계 (실연동)

CMS "포인트 내역"은 Firestore 포인트 원장(append-only)을 실집계한다. 출석으로 지급된 100P가 적립 레코드로 실시간 누적되고, 상단에 누적 적립 포인트·전체 원장 레코드 수·사용 차감·만료 소멸 집계 카드가 표시된다. 회수·차감도 삭제 없이 음수 레코드로 추가된다(10P = 1원).

CMS > 포인트 관리 > 포인트 내역 — 적립 +100P 레코드 실시간 누적 (이 문서 테스트 세션 발생분 포함 실데이터)
유형 컬럼은 현재 "적립"으로만 표기되고 사유(출석/룰렛 등) 구분은 노출되지 않는다. 사유 텍스트는 앱 활동내역에서 확인 가능. 상단 유형 필터의 "출석 적립" 옵션은 아직 미배선(목업).
8
CMS

운영자: 출석체크 설정 (포인트 관리)

CMS "출석체크 설정" 메뉴에서 일일 지급 포인트(100P), 연결 제휴몰(현재 쿠팡 단독), 5일 연속 보너스(이벤트 티켓 1장), 출석 인정 조건(제휴몰 방문 후 앱 복귀), 초기화 시각(00:00)을 관리한다. 상단에는 오늘 출석 회원·지급 포인트·5일 연속 달성·방문 전환율 집계 카드가 배치된다.

CMS > 포인트 관리 > 출석체크 설정 — 지급 설정 + 현재 적용 정책
데모 한계: 통계 4종은 실집계 미연동("운영 데이터 없음"), 지급 설정의 저장 버튼도 실제 저장으로 이어지지 않는 목업이다(룰렛 설정과 달리 미연동). 화면 좌측 메뉴 배지도 "확인중" 상태.

정책 · 규칙 (SoT: policyspec.html "출석 — 쿠팡 구경하기")

항목내용
기본 지급매일 앱을 거쳐 쿠팡 방문 + 30초 이상 체류 확인 시 100P. 하루 1회.
인정 조건"구경하기" 버튼을 먼저 눌러야 인정(그냥 쿠팡 앱만 열면 미인정). 쿠팡을 둘러본 뒤 앱으로 복귀해야 완료.
자정 경계앱으로 복귀한 시점의 날짜 기준.
환불 관계구매와 무관한 무상 보상 — 환불과 무관, 회수 없음.
5일 연속 보너스5일차도 매일 지급되는 100P는 동일, 추가로 이벤트 티켓 1장 지급(500P 보너스 없음). 이벤트 티켓 획득 독립 3종(출석·행운룰렛·온보딩) 중 하나. 누락 시 스트릭 1일부터 재시작.
2배 부스터출석 직후 일정 시간(현재 30분, 운영값) 내 경유 구매가 확정되면 추가 100P(총 200P). 일 1회. 추가분은 구매 확정 후 지급(환불 리스크 최소화). — 현재 데모 미구현.
이력 표시월~일 고정 달력이 아니라 오늘 기준 직전 7일 롤링으로 표시(미션이 연속 5일이라 요일과 무관 · DEVQA 켈빈 #5 확정). — 현재 데모는 고정형.
이벤트 티켓 규칙경품 응모 전용 · 받은 날부터 100일 만료(정책 기준). 일 5장 / 월 30장 공통 한도의 별개 경로(출석 5연속은 이부장 패키지와 독립 유지).
계정 병합카카오 연동 병합 시 이벤트성 무상지급분(출석·룰렛·온보딩)은 파밍 차단을 위해 합산에서 제외.

미결 · 데모-정책 불일치 (형님 확인 필요)

  • 이벤트 티켓 유효기간 표기 — 앱 출석 화면 2곳에 "유효기간 30일" 잔존. 확정 정책은 100일.
  • 주간 캘린더 표시 방식 — 데모는 월~일 고정형, 확정 정책은 직전 7일 롤링.
  • 2배 부스터 미구현 — 정책 확정 항목이나 데모 앱·CMS 모두 미반영.
  • CMS 부분 미연동 — 포인트 내역(원장 집계)은 실연동이나, 출석체크 설정 화면의 통계 4종·설정 저장은 목업. 티켓 내역의 "출석 체크" 발행유형 필터도 미배선.

참고 사항

  • 캡처 환경: 배포본 pricepick-demo.vercel.app/app에서 게스트 신규 계정(닉네임 자동 추천)으로 온보딩 후, 출석 플로우를 실제 수행(브릿지 이동 → 복귀 → 체류 검증 → 100P·티켓 지급)하며 캡처. CMS는 /cms 운영자 계정 로그인 후 캡처.
  • 배포본의 출석 메인 진입 상태는 "4일 연속(5일차)" 데모 시나리오로 고정되어 있어, 한 번의 실제 출석으로 5일 달성 흐름까지 확인된다. 상태 화면 3종(신규·당일 완료·리셋)은 배포본에 내장된 데모 상태 화면이다.
  • 브릿지 복귀 직후 뜨는 "쿠팡 구매 완료(구매 금액 입력)" 팝업은 경유 구매 시뮬레이터로, 출석과 별개 기능이라 이 문서에서는 다루지 않는다(→ 경유 구매 적립 스토리보드).
  • 지급 사유 라벨: 포인트 원장에는 쿠팡 구경 출석체크, 5일 달성 티켓은 출석 5일 연속 달성으로 기록된다.
