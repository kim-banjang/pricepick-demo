게스트

닉네임만으로 가입 — 미연동(게스트) 시작

로그인 없이 자동 추천 닉네임만 정하면 바로 시작한다. 가입 화면에 "이 기기에만 저장 · 앱 삭제/기기 변경 시 초기화" 경고가 노출되고, 카카오 연동으로 안전하게 보관할 수 있음을 안내한다.

가입 화면 · 닉네임 자동 추천 + 재설치·기기 변경 경고
2
게스트

연동 진입점 — 홈 배너 · 마이 회원정보 · 적립 조건 안내

미연동 상태에서는 앱 곳곳에서 연동을 유도한다. ① 홈 상단 "카카오 연동하고 혜택 받기" 배너(연동 계정에는 미노출), ② 마이 회원정보의 "카카오 미연동 · 연동하기" 행, ③ 적립 조건 안내의 "카카오 연동하러 가기" 링크. 어디서 눌러도 동일한 연동 플로우로 진입한다.

① 홈 배너 (미연동에만 노출)
② 마이 회원정보 · "카카오 미연동" + 연동하기
③ 적립 조건 안내 · 연동 유도 링크 (표기 이슈는 하단 미결 참조)

시스템

카카오 인증 — 카카오톡 앱 전환

연동하기를 누르면 카카오톡 앱 전환 안내와 함께 iOS 권한 팝업("카카오톡에서 프라이스픽 열기 허용")이 뜬다. 데모에서는 실제 OAuth 없이 이 화면에서 다음 단계로 진행한다. (화면 ID 03b-C)

카카오톡 이동 안내 + iOS 앱 전환 권한 팝업
4
분기

인증 결과 분기 — 신규 연결 vs 기존 계정 발견

해당 카카오 계정의 기존 데이터 유무에 따라 갈라진다. 신규 연결이면 보상 안내와 함께 연동이 완료되고, 기존 프라이스픽 데이터가 있는 카카오면 그 계정(티켓 잔량 포함)을 불러와 이어서 쓴다. 두 경로 모두 CTA를 누르는 순간 linked_kakao=true가 저장되고 홈으로 이동한다.

신규 연결 · 보상 지급

처음 연결하는 카카오 계정 (03b-B)

"카카오 연동 완료!" 화면에서 지금 지급되는 보상(이벤트 티켓 1장 + 가입 축하 200P + 영구 보관)과 이어서 받을 보상을 안내한다. CTA를 누르면 실제로 지급된다.

신규 연동 완료 · 보상 명세 + 시작하기
기존 계정 발견 · 데이터 복원

이미 사용하던 카카오 계정 (03b-A)

"이미 사용하신 계정이 있어요" — 기존 계정의 가입일·티켓 내역을 보여주고 그 계정으로 시작한다. 기기 변경·재설치 후 복원이 이 경로다. (썸네일 수치는 데모 정적 표시)

기존 계정 발견 · 티켓 잔량 확인 후 시작
구 "카카오 계정 충돌 선택" 화면(03-conflict)은 폐지. 양쪽 모두 적립을 보유한 경우 선택 화면 없이 자동 병합된다(6단계).
5
연동 완료

보상 지급 + 상태 전환 — 배너 숨김 · 활동내역 기록

연동 완료 시 users에 linked_kakao=true·kakao_login_id·kakao_nickname이 저장되고, 이벤트 티켓 1장(ticket_transactions, 유효 100일)과 200P(point_transactions + user_points 잔액)가 실제 지급된다. 홈의 연동 유도 배너는 즉시 사라지고, 활동내역에 지급 기록이 남는다.

연동 후 홈 · 배너 사라짐 + 200P 반영
활동내역 · "카카오 연동 가입 축하 +200P" + "이벤트 티켓 적립 +1장"
6
자동 병합

계정 통합 — 양쪽 모두 적립 보유 시 (03b-merge)

닉네임 계정에 적립이 있는 상태에서, 이미 적립 데이터가 있는 카카오로 연동하면 선택 화면 없이 자동 합산한다. 정상 적립 포인트·티켓은 합산 반영하고, 이벤트성 무상 지급분(출석·룰렛·온보딩 등)은 중복 방지를 위해 합산에서 제외한다.

계정 통합 완료 · 합산 요약 + 무상지급분 제외 안내
7
연동 회원

마이페이지 연동 상태 · 카카오 닉네임 동기화

연동 후 마이 회원정보 행이 "카카오 연동 · {카카오 닉네임} · {카카오 ID}" + "연동됨" 배지로 바뀐다. 닉네임 변경 화면에서는 "카카오 닉네임 가져오기"로 카카오 프로필 닉네임을 한 번에 입력할 수 있다(kakao_nickname은 앱 닉네임과 별개 필드, 중복 검사는 동일 적용).

마이 회원정보 · "장주원 · wfxpx65z4@kakao.com" + 연동됨
닉네임 변경 · 카카오 닉네임 가져오기(장주원) 노출
가져오기 클릭 → 입력란에 "장주원" 채워짐 + 저장 활성
8
CMS

운영자: 회원 목록 · 회원 상세의 연동 집계

CMS 회원 목록은 "카카오 연동 N명(%) / 게스트(미연동) N명(%)"을 실시간 집계하고, 첫 컬럼에 닉네임(카카오ID)·연동 상태 배지를 표기한다. 회원 상세에서는 연동 계정(카카오 ID), 가입 유형 "카카오 연동", 티켓 보유 현황을 확인·관리한다. 이 캡처의 회원(느긋한코알라)은 이 문서 제작 중 실제로 가입·연동한 테스트 계정이다.

CMS > 회원 목록 — 연동 17명(23.6%) / 게스트 55명(76.4%) 실집계 + 연동 상태 컬럼
CMS > 회원 상세 — 연동 계정(카카오 ID) · 가입 유형 "카카오 연동" · 이벤트 티켓 1장(연동 보상)

정책 요약 — 미연동(게스트) vs 카카오 연동 (SoT: 마스터 정책서 policyspec.html)

미결 · 불일치 — 형님 확인 필요 (이 문서 제작 중 실측으로 확인)

정책 불일치적립 조건 안내 화면의 대기기간 고정 표기 — 미연동(게스트) 상태에서도 "구매 확정 후 D+7 후 사용"으로 고정 표기된다(2단계 ③ 캡처). 정책은 미연동 D+30. 연동 상태에 따라 D+7/D+30 동적 표기 필요.
출처 충돌가입 축하 200P 지급 시점 — 마스터 정책서(계정 생애주기)는 "가입 즉시 200P + 이벤트 티켓(연동 후)"로 읽히는 반면, CMS 정책 텍스트·앱 실동작은 200P도 카카오 연동 시점에 지급(가입 시 0P 실측 확인). 어느 쪽이 확정인지 정리 필요.
버그CMS 회원 목록 검색·연동상태 필터 미동작 — 검색어 입력, "카카오 연동" 필터 선택 모두 목록에 반영되지 않음(실측). 목록은 최신 30명 고정 표시라 연동 회원 조회가 사실상 불가.
버그연동 직후 홈 포인트 이중 표시 — 연동 완료 직후 세션에서 홈 포인트가 400P로 표시(원장·서버 잔액은 200P 정상, 새로고침 시 정상 복귀). 로컬 증가분과 서버 재조회가 겹치는 표시 전용 버그.
기존 미결탈퇴 후 재가입 쿨타임 — CMS 약관 "동일 카카오 30일 제한" vs 앱 화면 "즉시 재가입 가능". 마스터 정책서 자체에 미결로 표기되어 있음(탈퇴 스토리보드 03과 공통 이슈).
데모 한계기존 계정 발견 화면의 수치는 정적 표시 — 실제 Kakao SDK가 없어 "기존 데이터 인식"은 데모에서 시뮬 불가. 4단계 기존 계정 카드(유저·12/5/1장)는 고정 데이터다.

참고 사항

  • 모든 앱 화면은 배포본(pricepick-demo.vercel.app/app)에서 헤드리스 브라우저로 게스트 계정을 실제 생성하고, 연동 진입 → 인증 → 완료 CTA까지 실클릭으로 진행한 뒤 캡처했다. CMS 화면은 운영자 계정으로 로그인해 캡처했다.
  • 연동 보상(이벤트 티켓 1장 + 200P)은 캡처 과정에서 실제로 Firestore에 기록됐고, 활동내역·CMS 회원 상세 캡처의 수치는 그 실데이터다.
  • 카카오 연동은 데모 시뮬이다: 실제 OAuth 없이 연동 완료 시 linked_kakao=true와 가상 kakao_login_id·kakao_nickname이 저장된다. 재연동 시 기존 값은 유지된다.
  • 버전 번호는 쓰지 않는다 — 기준은 항상 pricepick-demo 최신 배포본.
