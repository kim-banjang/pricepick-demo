활동내역 — 당첨 보상 기록

당첨된 이벤트 티켓은 활동내역에 "이벤트 티켓 적립 +1장"으로 장 단위 기록되고, 포인트 당첨은 포인트 원장에 "행운 룰렛 당첨"으로 기록된다. 티켓은 경품 응모에서만 사용할 수 있다.

CMS

운영자: 행운룰렛 설정 — 6칸 유형·값 편집 (실연동)

CMS "행운룰렛 설정"에서 6칸 각각의 유형(꽝 / 이벤트티켓 / 포인트)과 값을 편집하고 저장하면 settings/roulette 문서에 기록되고, 앱은 룰렛 진입 때마다 이 설정을 읽어 휠 렌더링과 지급에 그대로 사용한다. 확률·배분이 정책상 미결이라 운영자가 직접 정하는 구조다.

항목내용
기본출석과 무관한 독립 상시 이벤트. 하루 1회 참여, 매일 자정 초기화.
보상포인트·이벤트 티켓을 확률로 지급. 이벤트 티켓의 정식 획득 경로 중 하나(출석 5연속·온보딩과 독립 3종).

이벤트 티켓 한도전 경로 합산 일 5장 / 월 30장. 초과 시 당첨돼도 미지급(0장) 처리.
티켓 사용·만료경품 응모(추첨) 전용. 받은 날부터 100일 만료(정책 기준).

미결 · 확인 필요 (형님 확인)

  • 6칸 기본 배분값 확정 대기 — 현재 시드: 포인트 100·200P / 이벤트 티켓 1·2장 / 꽝 2칸. 확정 전까지 시드값으로 운영 중이며 임의 확정하지 않음.
  • 보상 확률·배분 정책 미결(policyspec 명시) — 균등 1/6 외에 칸별 확률 차등이 필요하면 정책 결정 + 기능 추가 필요. 정책서 권고: 이벤트 티켓 발행을 어뷰징 한도(일 5/월 30) 안에서 관리.
  • 이벤트 티켓 유효기간 표기 — 정책은 100일이나 앱 일부 화면에 "유효기간 30일" 표기 잔존(출석 화면 쪽, 룰렛 안내 문구에는 기간 표기 없음).
  • 하루 1회 판정의 서버화 — 데모는 기기 localStorage 기준. 실서비스 전 서버 판정 전환 필요.

참고 사항

  • 캡처 환경: 배포본 pricepick-demo.vercel.app/app 게스트 신규 계정으로 룰렛을 실제 스핀하며 캡처(당첨 보상 실지급 확인). 결과 3종은 모두 실제 추첨 결과다 — 한도 초과 화면은 같은 계정이 당일 이벤트 티켓 5장을 채운 뒤 티켓 칸에 다시 당첨되어 나온 실화면.
  • CMS 화면은 /cms 운영자 계정 로그인 후 캡처. "행운룰렛 설정"은 Firestore settings/roulette 실연동(저장 시 앱 즉시 반영)이며, 좌측 메뉴 배지는 "확인중" 상태.
  • 앱 룰렛은 진입할 때마다 CMS 설정을 재조회하므로, CMS에서 배분을 바꾸면 별도 배포 없이 다음 진입부터 반영된다. CMS 문서가 없거나 형식이 깨지면 앱은 내장 시드값으로 대체 렌더링한다.
